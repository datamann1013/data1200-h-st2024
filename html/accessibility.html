welcome to accessibility hell

This page covers how we made our site accessible for all users by making coding choices for universal design.

We chose the font Verdana, with sans-serif as a backup. The justification for this choice was based on it's legibility and accessible design by being simple and unembellished. The font should clearly differentiate the letters well even in small text sizes. We've avoided fonts featuring “imposter letter shapes”. These include characters that are visually similar, but not the same, for example uppercase and lowercase Is, lowercase Ls, and the number 1 (Iil1). We've ensured that the font size is in a relative unit, we used rem, to scale it properly based on individual user needs. This is because using absolute units, like pixels, limits the option to increase or decrease text size without zooming in on everything else on the page. All paragraphs use 1.5 line-height to make the text easier to read. letter-spacing is 0.04rem

General layout, clear headers and high-resolution images. Understandable: Implicitly imply the content without needing to read everything. Keeping headers in correct order and using semantic tags.

Choices and justification Lightmode and darkmode = contrast on both! darkmode links are lighter than lightmode links while still being assosiated to hyperlinks

PNG, WebP, JPEG Backup image formatting nescessary? Multiple versions of one image to support different screen sizes

We've made sure to include a descriptive alt text for visually impaired users. Images and icons should have this

Screen readers (aria-label). Tabindex for non-mouse users

We started testing the accessibility of our sites by generating built in Lighthouse reports in Google Chrome. The first test gave an overall score of 22/25 and highlighted missing labels in forms and problematic lists. We checked the code to find the source of these problems.
