A short list of benefits of Git and Github in VS code

Github can make collaborations easier

When working in a team, GitHub is often used to share and collaborate on code. With GitHub tools built into VS Code, you can review code changes, make comments, and open pull requests directly from the editor. This can make it easier to collaborate with others because you don’t need to switch to a web browser. You can work with your team in real-time without leaving the editor, and all the GitHub features are right at your fingertips

Git branches

Branches are a key part of using Git, allowing you to work on different features or fixes without affecting the main codebase. In VS Code, you can create new branches, switch between them, and merge changes back into the main branch with just a few clicks when needed. This can help to keep your work organized and prevent conflicts from appearing between different pieces of code. Plus, you can easily see which branch you’re working on at any time.
