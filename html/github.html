A short list of benefits of Git and Github in VS code

Github can make collaborations easier

When working in a team, collaboration and code reviews are important to keep the code quality high and to make sure that everyone´s contributions are in sync. Github together with Visual Studio Code can offer an easy way to achieve this by giving direct access to GitHub´s collaboration tools inside Visual Studio Code. This can let developers review code, create and manage pull requests, leave comments, and work together in real-time. It can make it easier to collaborate with others because there is no need to constantly switch to a web browser. You can stay within Visual Studio Code and still use all of GitHub´s tools.

Git branches

Branches are an importan part of using Git becuase they let developers work on different features without affecting the main codebase. In Visual Studio Code, you can create new branches, switch between them, and merge them back into the main branch when needed. By creating branches, developers can work and test their changes separately form the main branch. This can help to keep the work organized and prevent conflicts from happening between different pieces of code. Each branch can focus on a specific task or feature, making it easier to review and merge changes once the work is complete.
